# Legend — ตัวย่อทั้งหมด

พจนานุกรมตัวย่อฉบับเต็มของ prayat อ่านเมื่อเจอโค้ดที่ไม่คุ้น หรือเมื่อผู้ใช้ถามว่ามีโค้ดอะไรบ้าง

## วิธีใช้

พิมพ์ `โค้ด: เนื้อหา` — ทุกอย่างหลัง `:` คือหัวข้อ/เนื้อหา เช่น `s: ข่าวที่อยากให้สรุป`
ไม่ใส่โค้ดก็ได้ — ถ้าโหมดเปิดอยู่ Claude จะตอบไทยกระชับให้อยู่ดี

## โค้ดหลัก

| โค้ด | ความหมาย | ผลลัพธ์ |
|------|----------|---------|
| `s:` | สรุป | bullet สั้น เฉพาะใจความ |
| `t:` | แปล (เดาทิศ TH↔EN) | คำแปลล้วน ไม่มีคำอธิบายเกิน |
| `e:` | อธิบาย | เข้าใจง่าย สั้น |
| `w:` | ทำไม / เหตุผล | เหตุผลเป็นข้อ |
| `h:` | ขั้นตอนวิธีทำ | ลำดับมีหมายเลข |
| `cmp:` | เปรียบเทียบ | ข้อดี-ข้อเสีย |
| `c:` | เขียนโค้ด | โค้ด + โน้ตสั้น |
| `cc:` | เขียนโค้ดล้วน | โค้ดอย่างเดียว ไม่มีคำอธิบาย |
| `f:` | แก้บั๊ก/แก้โค้ด | โค้ดที่แก้ + สาเหตุ 1–2 บรรทัด |
| `r:` | รีวิวโค้ด | เฉพาะประเด็นสำคัญ |
| `i:` | ปรับปรุง / refactor | เวอร์ชันที่ดีขึ้น + เหตุผลสั้น |

## ตัวขยาย (ต่อท้ายโค้ด หรือใช้เดี่ยว)

| ตัวขยาย | ผล | ตัวอย่าง |
|---------|----|----------|
| `+` | ละเอียดขึ้น | `s+:`, `e+:` |
| `-` | สั้นลงอีก | `r-:`, `e-:` |
| `en` | ตอบเป็นอังกฤษ (output ถูกกว่า) | `e en:` |
| `?` | สั้นสุด: ใช่/ไม่ + เหตุผล 1 บรรทัด | `?: ใช้ index ตรงนี้คุ้มไหม` |

## ตัวอย่างจริง

**สรุป** — `s: <บทความยาว>` → bullet 3–5 ข้อ ไม่มีเกริ่น
**แก้โค้ด** — `f: def add(a,b): return a-b` → โค้ดที่แก้ + "สาเหตุ: ใช้ - แทน +"
**โค้ดล้วน** — `cc: ฟังก์ชัน Python เช็กจำนวนเฉพาะ` → โค้ดอย่างเดียว
**ถามสั้น** — `?: useMemo กับ useCallback แทนกันได้ไหม` → "ไม่ได้ — useMemo คืน value, useCallback คืน function"
**ตอบอังกฤษ** — `e en: closure ใน JavaScript` → คำอธิบายสั้นภาษาอังกฤษ

## เพิ่มโค้ดเอง

เพิ่มแถวในตาราง "โค้ดหลัก" รูปแบบ `โค้ด | ความหมาย | ผลลัพธ์` แล้ว Claude จะใช้ตามนั้น
หลักการ: สั้น (1–3 ตัว), จำง่าย, ไม่ชนของเดิม, ลงท้าย `:` เสมอ

> หมายเหตุ: โค้ดทำงานเต็มที่เมื่อโหมดเปิด (กติกาถูกฉีดเข้าทุก turn) ถ้าโหมดปิด โค้ดยังพอใช้ได้แต่โมเดลอาจไม่รู้จักครบ
